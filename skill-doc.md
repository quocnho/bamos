## AI Sprint Cadence

| Day | Focus | Activity |
|-----|-------|----------|
| **Day 1** | Architecture | AI generates scaffold, interfaces, DB schema. Human reviews. |
| **Day 2** | Core Build | AI + human pair-program core features. Highest output day. |
| **Day 3** | Integration | AI handles integrations, edge cases, runs test suites. |
| **Day 4** | Polish | AI generates docs, refactors. Human reviews all PRs. |
| **Day 5** | Ship | Demo, retrospective, CI/CD deploys to production. |

**5-day AI sprint = 2-4 week traditional sprint** (3-5× acceleration)

## Key Principles

1. **Read /plan/ before coding** — Every line of code maps to a user story
2. **AI generates, human reviews** — AI writes 70-80%, human reviews 100%
3. **Conventional Commits** — `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`
4. **Sprint = 5 days max** — If it takes longer, split the story
5. **Deploy every sprint** — Working software > documentation
