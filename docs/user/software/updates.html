🔄 Cập nhật và Rollback trên BamOS

BamOS sử dụng công nghệ OCI images và RPM-OSTree cho phép cập nhật nguyên tử (atomic updates) và rollback dễ dàng. Hệ thống luôn an toàn — nếu bản cập nhật có lỗi, bạn có thể quay lại phiên bản trước chỉ với một lệnh.

📹 Hướng dẫn cập nhật và rollback trên BamOS (sẽ thêm sau)

💡 Cập nhật nguyên tử là gì?

Không giống các bản phân phối truyền thống (cập nhật từng gói một), BamOS tải toàn bộ hệ thống mới dưới dạng image. Quá trình cập nhật diễn ra trong nền, không ảnh hưởng đến hệ thống đang chạy. Bạn chỉ cần khởi động lại để áp dụng.

🔄 Cập nhật hệ thống

Cách 1: Dùng Software Center

Cả GNOME Software và KDE Discover đều hiển thị thông báo khi có cập nhật. Bạn chỉ cần nhấn "Update All".

Cách 2: Dùng CLI (Khuyên dùng)

# Kiểm tra cập nhật
sudo bazzite-rollback-helper update

# Hoặc dùng rpm-ostree trực tiếp
rpm-ostree update

# Sau khi tải xong, khởi động lại
systemctl reboot

Cập nhật Flatpak

# Cập nhật tất cả Flatpak
flatpak update

# Cập nhật một ứng dụng cụ thể
flatpak update org.gimp.GIMP

Cập nhật Distrobox containers

# Cập nhật một container
distrobox upgrade ubuntu

# Cập nhật tất cả containers
distrobox upgrade --all

🔍 Kiểm tra trạng thái

# Xem thông tin deployment hiện tại
rpm-ostree status

# Kết quả sẽ hiển thị:
# ● bamos:fedora-39 (deployed)
#   Layer packages: steam lutris
#   Initramfs: 2026-01-15
#   Timestamp: 2026-01-15T12:00:00Z

# Xem version BamOS
cat /etc/os-release | grep VERSION

⏪ Rollback (Quay lại phiên bản cũ)

Một trong những tính năng mạnh nhất của BamOS là khả năng rollback dễ dàng. RPM-OSTree giữ lại các deployment trước đó, cho phép bạn quay lại bất kỳ lúc nào.

# Xem danh sách các phiên bản đã cài
rpm-ostree status

# Rollback về phiên bản trước
sudo rpm-ostree rollback

# Hoặc chọn index cụ thể
sudo rpm-ostree deploy 1

# Khởi động lại để áp dụng
systemctl reboot

⚠️ Quan trọng

Sau khi rollback, mọi gói đã layer (cài qua rpm-ostree install) sẽ được giữ nguyên. Tuy nhiên, các thay đổi trong /etc có thể cần cấu hình lại nếu rollback xuống phiên bản cũ hơn nhiều.

📦 Quản lý Deployment

Lệnh Mô tả
rpm-ostree status Xem danh sách deployments
rpm-ostree rollback Quay về deployment trước
rpm-ostree deploy <index> Chuyển sang deployment theo index
rpm-ostree cleanup -m Xóa các deployment cũ (trừ 2 bản gần nhất)
rpm-ostree upgrade Cập nhật lên bản mới nhất
rpm-ostree reload Tải lại cấu hình (không cần reboot)

📥 Quản lý gói đã layer

# Xem danh sách gói đã layer
rpm-ostree status

# Thêm gói
sudo rpm-ostree install htop

# Gỡ gói
sudo rpm-ostree uninstall htop

# Cập nhật gói đã layer (tự động khi update)

📝 Layer packages vs Flatpak

Layer packages (rpm-ostree install) được cài vào hệ thống gốc và yêu cầu reboot. Flatpak không cần reboot. Nguyên tắc: ưu tiên Flatpak, chỉ layer package khi thực sự cần.

🛠 Bazzite Rollback Helper

BamOS tích hợp bazzite-rollback-helper — công cụ giúp quản lý cập nhật và rollback dễ dàng hơn:

# Kiểm tra và cập nhật
sudo bazzite-rollback-helper update

# Rollback
sudo bazzite-rollback-helper rollback

# Kiểm tra trạng thái
sudo bazzite-rollback-helper status

🖥 Chọn phiên bản khi khởi động

Khi khởi động, bạn có thể chọn phiên bản BamOS muốn dùng từ GRUB boot menu:

  1. Khi thấy màn hình GRUB, nhấn Esc hoặc giữ Shift
  2. Chọn "BamOS (previous deployment)"
  3. Nhấn Enter để boot vào phiên bản cũ

🛠 Xử lý sự cố

📝 Cập nhật thất bại

# Kiểm tra log
sudo journalctl -u rpm-ostreed -f

# Thử lại với verbose
rpm-ostree update --verbose

# Rollback và thử lại sau
sudo rpm-ostree rollback

⚠️ Dung lượng ổ cứng

Mỗi deployment RPM-OSTree chiếm khoảng 5-10GB. Nếu ổ cứng đầy, hãy dọn dẹp: sudo rpm-ostree cleanup -m.
