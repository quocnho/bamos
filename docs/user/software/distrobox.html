📦 Distrobox

Distrobox là công cụ mạnh mẽ giúp bạn tạo và quản lý các container chạy bản phân phối Linux khác nhau, tích hợp sâu với hệ thống BamOS host. Đây là giải pháp lý tưởng để dùng các gói phần mềm từ nhiều distro khác nhau.

📹 Hướng dẫn sử dụng Distrobox trên BamOS (sẽ thêm sau)

💡 Distrobox hoạt động thế nào?

Distrobox dùng Podman hoặc Docker để tạo container, sau đó tự động tích hợp container với host — chia sẻ thư mục home, thiết bị, âm thanh, GPU và D-Bus. Ứng dụng cài trong container xuất hiện ngay trong menu ứng dụng!

📥 Cài đặt

Distrobox được cài đặt sẵn trên BamOS. Nếu chưa có, bạn có thể cài bằng:

# Cài Distrobox
sudo rpm-ostree install distrobox

# Hoặc qua pip
pip install distrobox

🆕 Tạo Container

Tạo container Ubuntu

# Tạo container tên "ubuntu"
distrobox create --name ubuntu --image ubuntu:latest

# Vào container
distrobox enter ubuntu

# Cài gói (bên trong container)
sudo apt update
sudo apt install git curl build-essential

Tạo container Arch Linux

# Tạo container Arch
distrobox create --name arch --image archlinux:latest

# Vào container Arch
distrobox enter arch

# Cài gói qua pacman
sudo pacman -Sy base-devel git
# Cài yay (AUR helper)
git clone https://aur.archlinux.org/yay.git
cd yay && makepkg -si

Tạo container Fedora

# Tạo container Fedora
distrobox create --name fedora --image fedora:39

# Vào container
distrobox enter fedora
sudo dnf install neovim htop

⌨ Các lệnh Distrobox

Lệnh Mô tả
distrobox create --name <tên> --image <image> Tạo container mới
distrobox enter <tên> Vào container
distrobox list Liệt kê các container
distrobox stop <tên> Dừng container
distrobox rm <tên> Xóa container
distrobox update Cập nhật container
distrobox upgrade --all Cập nhật tất cả containers
distrobox init Khởi tạo container hiện tại

🔄 Tích hợp với Host

Sau khi tạo container và cài ứng dụng bên trong, Distrobox tự động tạo file .desktop cho từng ứng dụng, giúp chúng xuất hiện trong menu ứng dụng của BamOS.

# Cài GIMP trong container Ubuntu
distrobox enter ubuntu
sudo apt install gimp
# Sau khi cài, GIMP sẽ xuất hiện trong menu ứng dụng!

# Chạy lệnh từ host vào container
distrobox enter ubuntu -- gimp

📝 Mẹo: Export ứng dụng

Mặc định Distrobox tự động export ứng dụng. Bạn có thể kiểm soát bằng lệnh: distrobox-export --app <tên> hoặc distrobox-export --bin /usr/bin/htop --export-path ~/.local/bin

📂 Chia sẻ dữ liệu

Container Distrobox tự động mount thư mục $HOME của bạn vào container. Điều này có nghĩa là:

  • 📁 Tất cả file trong ~/Documents, ~/Downloads,... đều có sẵn trong container
  • ⚙ Cấu hình ~/.ssh, ~/.gitconfig được dùng chung
  • 🖥 GPU và thiết bị USB được chia sẻ tự động

🎯 Các trường hợp sử dụng

1. Phát triển phần mềm

# Tạo container cho từng dự án
distrobox create --name project-a --image fedora:39
distrobox enter project-a
# Cài dependencies cho dự án A
sudo dnf install golang postgresql-devel

2. Dùng gói AUR

distrobox enter arch
yay -S google-chrome visual-studio-code-bin

3. Thử nghiệm distro

# Thử nghiệm OpenSUSE
distrobox create --name opensuse --image opensuse/tumbleweed
distrobox enter opensuse
sudo zypper install neofetch
neofetch

⚙ Tùy chỉnh nâng cao

# Tạo container với các tùy chọn
distrobox create \
  --name dev \
  --image fedora:39 \
  --additional-flags "--security-opt label=disable" \
  --home ~/distrobox/dev-home

# Export script ra host
distrobox-enter dev -- htop

# Chạy ứng dụng GUI từ container
distrobox enter dev -- code

⚠️ Lưu ý

Không nên dùng Distrobox để chạy các ứng dụng cần hiệu năng đồ họa cao (game 3D). Hãy dùng Flatpak hoặc RPM-OSTree cho các trường hợp đó.

🛠 Xử lý sự cố

📝 Container không vào được

# Kiểm tra trạng thái
podman ps -a

# Khởi động lại container
distrobox stop ubuntu
distrobox enter ubuntu

# Xem log
podman logs ubuntu
