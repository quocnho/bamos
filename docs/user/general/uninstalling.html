🗑 Gỡ Cài Đặt BamOS

Hướng dẫn này giúp bạn gỡ bỏ BamOS khỏi máy tính và khôi phục lại hệ điều hành trước đó (Windows hoặc distro Linux khác).

⚠️ Cảnh báo quan trọng

Sao lưu dữ liệu trước khi tiếp tục! Gỡ cài đặt hệ điều hành sẽ xóa toàn bộ dữ liệu trên các phân vùng tương ứng. Hãy đảm bảo bạn đã sao lưu tất cả dữ liệu quan trọng vào ổ cứng ngoài hoặc cloud storage.

💡 Bạn có chắc muốn gỡ?

BamOS có tính năng rollback — nếu bạn gặp vấn đề sau khi cập nhật, hãy thử rollback về generation trước đó thay vì gỡ hoàn toàn. Khởi động lại và chọn generation cũ trong GRUB.

Các Tình Huống Gỡ Cài Đặt

  • Dual-boot với Windows: Xóa phân vùng BamOS và sửa bootloader Windows.
  • Dual-boot với Linux khác: Xóa phân vùng BamOS và cập nhật GRUB.
  • BamOS là HĐH duy nhất: Cài lại Windows hoặc distro khác từ USB.

Bước 1: Sao Lưu Dữ Liệu

# Sao lưu thư mục home vào ổ cứng ngoài
rsync -avh /home/your-username/ /run/media/your-username/backup/bamos-home/

# Hoặc dùng tar
tar -czvf bamos-backup.tar.gz /home/your-username/

Bước 2: Xác Định Phân Vùng BamOS

# Kiểm tra phân vùng ổ cứng
lsblk

# Xem chi tiết
sudo fdisk -l

# Kết quả sẽ hiển thị các phân vùng, ví dụ:
# /dev/nvme0n1p1 - EFI partition
# /dev/nvme0n1p2 - Windows (NTFS)
# /dev/nvme0n1p3 - BamOS root (ext4)
# /dev/nvme0n1p4 - BamOS /home (ext4)

Bước 3: Gỡ BamOS trong Dual-Boot với Windows

Sửa bootloader bằng Windows Recovery USB

# Khởi động từ Windows Recovery USB
# Chọn "Repair your computer" → "Troubleshoot" → "Advanced options" → "Command Prompt"

# Khôi phục bootloader Windows
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd

Xóa phân vùng BamOS từ Windows

# Mở Disk Management (diskmgmt.msc)
# Hoặc dùng diskpart trong Command Prompt:
diskpart
list disk
select disk 0
list partition
select partition X  # X là số phân vùng BamOS
delete partition override
exit

Mở rộng phân vùng Windows

Sau khi xóa phân vùng BamOS, bạn có thể mở rộng phân vùng Windows để tận dụng dung lượng trống:

# Trong Disk Management, chuột phải phân vùng Windows
# Chọn "Extend Volume" và làm theo hướng dẫn

Bước 4: Gỡ BamOS trong Dual-Boot với Linux Khác

# Khởi động vào distro Linux còn lại
# Xóa phân vùng BamOS
sudo umount /dev/nvme0n1p3  # phân vùng root của BamOS
sudo umount /dev/nvme0n1p4  # phân vùng home nếu có

sudo fdisk /dev/nvme0n1
# Dùng lệnh 'd' để xóa phân vùng, 'w' để ghi thay đổi

# Cập nhật GRUB
sudo update-grub
# hoặc
sudo grub-mkconfig -o /boot/grub/grub.cfg

Bước 5: Cài Lại Từ Đầu (BamOS là HĐH duy nhất)

Nếu BamOS là hệ điều hành duy nhất và bạn muốn chuyển sang Windows hoặc distro khác:

  1. Tạo USB boot cho HĐH mới (Windows / Ubuntu / Fedora / ...)
  2. Khởi động từ USB
  3. Trong quá trình cài đặt, chọn xóa tất cả phân vùng hoặc tùy chỉnh phân vùng thủ công
  4. Xóa các phân vùng BamOS và tạo phân vùng mới cho HĐH mới
  5. Hoàn tất cài đặt

Khắc Phục Sự Cố Thường Gặp

Sự cốNguyên nhânGiải pháp
Không boot được sau khi xóa phân vùng Bootloader bị hỏng Dùng Windows Recovery USB để sửa bootloader
Windows không khởi động EFI partition thiếu entry Windows Dùng bcdedit hoặc EasyBCD để thêm lại
GRUB vẫn hiển thị BamOS GRUB chưa được cập nhật Chạy sudo update-grub từ distro còn lại
Phân vùng EFI còn file BamOS File bootloader chưa được xóa Xóa thư mục /boot/efi/EFI/bamos nếu còn

📝 Ghi chú cuối

Sau khi gỡ cài đặt, bạn có thể quay lại BamOS bất cứ lúc nào bằng cách tải ISO mới nhất từ trang ⬇ Download và cài đặt lại. Chúng tôi hy vọng bạn sẽ quay lại! ❤️
