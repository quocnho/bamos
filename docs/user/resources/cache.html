💾 Binary Cache (Cachix)

Binary cache là một trong những tính năng mạnh mẽ nhất của hệ sinh thái Nix. Nó cho phép lưu trữ các gói đã được build sẵn dưới dạng nhị phân (binary), giúp người dùng khác tải về mà không cần tự build từ mã nguồn.

⏱ Tác động: Nhờ binary cache, thời gian cài đặt BamOS giảm từ vài giờ (build từ source) xuống còn vài phút (tải binary).

Cachix là gì?

Cachix là dịch vụ lưu trữ binary cache cho Nix, tương tự Docker Hub nhưng dành cho Nix. BamOS sử dụng Cachix để phân phối các gói build sẵn đến người dùng.

Cache công cộng của BamOS

BamOS có một binary cache công cộng trên Cachix, được tự động cập nhật mỗi khi có thay đổi trên nhánh main của repository. Bạn không cần làm gì thêm — cache đã được cấu hình sẵn trong flake.

# Cache công cộng của BamOS bamos.cachix.org

Cách sử dụng cache

Tự động (đã cấu hình sẵn)

Nếu bạn đang dùng BamOS, cache đã được tự động cấu hình. Bạn có thể kiểm tra:

# Xem cấu hình cache hiện tại nix show-config | grep substituters # Kết quả sẽ bao gồm: # https://bamos.cachix.org

Thủ công (cho người dùng NixOS khác)

Nếu bạn muốn sử dụng cache BamOS trên một hệ thống NixOS khác:

# Thêm Cachix repository nix-shell -p cachix --run "cachix use bamos" # Hoặc thêm vào cấu hình Nix: # nix.settings.substituters = [ "https://bamos.cachix.org" ]; # nix.settings.trusted-public-keys = [ "bamos.cachix.org-1:..." ];

Push gói lên cache

Nếu bạn là maintainer hoặc có quyền truy cập vào cache BamOS, bạn có thể push các gói đã build lên cache:

Yêu cầu

  • Tài khoản Cachix (đăng ký tại app.cachix.org)
  • Auth token từ Cachix
  • Quyền truy cập vào cache bamos

Các bước thực hiện

# 1. Cài đặt cachix nix profile install nixpkgs#cachix # 2. Xác thực cachix authtoken <token-của-bạn> # 3. Push gói lên cache nix build .#package-name cachix push bamos result # Push nhiều gói cùng lúc cachix push bamos --watch-store
⚠️ Bảo mật: Không bao giờ commit auth token vào repository. Sử dụng GitHub Secrets hoặc environment variables.

CI/CD tự động push cache

BamOS sử dụng GitHub Actions để tự động build và push cache mỗi khi có commit mới. Dưới đây là ví dụ workflow:

# .github/workflows/cache.yml name: "Build & Push Cache" on: push: branches: [main] jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: DeterminateSystems/nix-installer-action@v4 - uses: DeterminateSystems/magic-nix-cache-action@v2 - uses: cachix/cachix-action@v14 with: name: bamos authToken: '${{ secrets.CACHIX_AUTH_TOKEN }}' - run: nix build .#iso-desktop - run: cachix push bamos result

Kiểm tra cache

Bạn có thể kiểm tra trạng thái và thống kê của cache BamOS:

# Xem thông tin cache cachix info bamos # Kết quả: # Name: bamos # Size: ~XX GB # Objects: XXXX # Last updated: ...

Cache địa phương (Local Cache)

Nix cũng lưu cache địa phương trên máy của bạn. Bạn có thể quản lý nó:

# Xem dung lượng cache du -sh /nix/store # Dọn dẹp cache cũ (garbage collection) nix store gc # Dọn dẹp triệt để hơn nix store gc --option max-freed 100000000000 # 100 GB
💡 Mẹo: Chạy nix store gc định kỳ để giải phóng dung lượng ổ cứng. Các deployment cũ của OSTree cũng có thể được dọn bằng sudo ostree admin cleanup.

So sánh tốc độ: có cache vs không cache

Gói Không cache (build từ source) Có cache (tải binary)
BamOS Desktop ISO 45–60 phút 5–10 phút
BamOS Gaming ISO 60–90 phút 8–15 phút
Gói đơn (vd: neofetch) 2–3 phút 5–10 giây
Python với dependencies 10–20 phút 30–60 giây
