🔄 Phương pháp Cài đặt Khác

Ngoài phương pháp cài đặt tiêu chuẩn qua USB, BamOS còn hỗ trợ nhiều cách cài đặt khác phù hợp với nhu cầu đa dạng của người dùng.

🪟 Cài song song với Windows (Dual-Boot)

Yêu cầu

Windows phải được cài ở chế độ UEFI (không phải Legacy BIOS). Kiểm tra bằng cách vào msinfo32 → mục "BIOS Mode" phải là "UEFI".

Chuẩn bị phân vùng

  1. Vào Windows → Disk Management
  2. Thu nhỏ phân vùng Windows (Shrink Volume) để tạo dung lượng trống tối thiểu 30GB
  3. Để trống phân vùng đó (chưa format) — trình cài đặt sẽ tự xử lý

Các bước cài đặt

  1. Khởi động từ USB BamOS (như hướng dẫn trong 📖 Guide)
  2. Khi đến bước phân vùng, chọn "Cài đặt cùng với Windows" hoặc chọn thủ công phân vùng trống
  3. Boot loader sẽ tự động phát hiện Windows để thêm vào GRUB

Chọn HĐH khi khởi động

Sau khi cài đặt xong, mỗi lần khởi động bạn sẽ thấy menu GRUB cho phép chọn:

  • BamOS — Khởi động vào BamOS
  • Windows Boot Manager — Khởi động vào Windows

Mẹo

Bạn có thể thay đổi thời gian chờ mặc định và HĐH mặc định trong GRUB bằng cách sửa file /etc/default/grub.

💻 Cài đặt trên Máy Ảo (Virtual Machine)

BamOS có thể chạy trong máy ảo để dùng thử mà không ảnh hưởng đến hệ thống chính.

VMWare Workstation / VirtualBox

  1. Tạo máy ảo mới với các thông số:
    • Hệ điều hành: Linux (Fedora 64-bit)
    • RAM: tối thiểu 4GB
    • Ổ cứng: tối thiểu 30GB
    • Bộ xử lý: 2 cores
  2. Gắn file ISO BamOS vào ổ CD/DVD ảo
  3. Khởi động máy ảo và cài đặt như bình thường

GNOME Boxes

Cách đơn giản nhất trên Linux:

  1. Mở GNOME Boxes
  2. Chọn "Create a Virtual Machine"
  3. Chọn file ISO BamOS
  4. Điều chỉnh dung lượng và RAM rồi nhấn "Create"

Ghi chú

Hiệu năng đồ họa trong máy ảo sẽ thấp hơn so với cài thật. Bạn nên cài đặt Guest Additions/SPICE để cải thiện trải nghiệm.

🔧 Cài qua Distrobox / Toolbox

Nếu bạn đang dùng một bản phân phối Linux khác, bạn có thể dùng Distrobox để chạy các công cụ BamOS trong container:

# Tạo container BamOS
$ distrobox create --name bamos --image ghcr.io/quocnho/bamos:latest

# Vào container
$ distrobox enter bamos

Xem thêm tại trang 📦 Distrobox.

📡 Cài qua mạng (PXE Boot)

Dành cho quản trị viên muốn triển khai BamOS trên nhiều máy cùng lúc qua mạng nội bộ:

Cấu hình PXE yêu cầu kiến thức về DHCP, TFTP và HTTP. Chi tiết sẽ được cập nhật sau.

So sánh các phương pháp

Phương pháp Độ khó Hiệu năng Phù hợp cho
USB (tiêu chuẩn) Dễ Tối đa Đa số người dùng
Dual-Boot Trung bình Tối đa Người cần cả Windows + Linux
Máy ảo Dễ Thấp hơn Dùng thử, phát triển
PXE/Netboot Khó Tối đa Triển khai hàng loạt
Distrobox Dễ Native Dùng công cụ trên distro khác
