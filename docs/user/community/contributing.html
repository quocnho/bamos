🤝 Hướng dẫn đóng góp cho BamOS

BamOS là một dự án mã nguồn mở và chúng tôi luôn chào đón mọi đóng góp từ cộng đồng. Dù bạn là lập trình viên dày dạn hay người mới bắt đầu, đều có nhiều cách để bạn tham gia và giúp BamOS ngày càng tốt hơn.

Các cách đóng góp

  • Viết code — Thêm tính năng mới, sửa lỗi, tối ưu hiệu năng
  • Cải thiện tài liệu — Sửa lỗi chính tả, viết hướng dẫn mới, dịch thuật
  • Báo lỗi — Phát hiện và báo cáo lỗi qua GitHub Issues
  • Thảo luận — Tham gia cộng đồng, giúp đỡ người dùng khác
  • Xây dựng gói — Đóng góp gói phần mềm mới vào overlay

Quy trình đóng góp (GitHub Flow)

1. Fork repository

Truy cập github.com/quocnho/bamos và nhấn nút Fork ở góc trên bên phải.

2. Clone fork về máy

git clone https://github.com/<tên-bạn>/bamos.git cd bamos

3. Thêm upstream remote

git remote add upstream https://github.com/quocnho/bamos.git # Kiểm tra remote git remote -v

4. Tạo nhánh mới

Tạo một nhánh riêng cho tính năng hoặc bản sửa lỗi của bạn:

# Luôn cập nhật nhánh main từ upstream trước git checkout main git pull upstream main # Tạo nhánh mới git checkout -b feat/ten-tinh-nang # hoặc git checkout -b fix/ten-loi

Sử dụng tiền tố feat/ cho tính năng mới, fix/ cho sửa lỗi, docs/ cho tài liệu.

5. Thực hiện thay đổi

Sửa mã nguồn, thêm tệp mới, hoặc cập nhật tài liệu. Sau đó kiểm tra kỹ:

# Kiểm tra thay đổi git status git diff # Chạy thử (nếu có) nix flake check

6. Commit theo Conventional Commits

BamOS sử dụng Conventional Commits để tự động quản lý phiên bản và changelog.

git add . git commit -m "feat: thêm gói libreoffice vào overlay" # Một số ví dụ commit hợp lệ: # feat: thêm tính năng mới # fix: sửa lỗi # docs: cập nhật tài liệu # refactor: tái cấu trúc code # chore: thay đổi công cụ build, dependencies # test: thêm hoặc sửa test

7. Push lên fork

git push origin feat/ten-tinh-nang

8. Tạo Pull Request

Trên GitHub, vào repository fork của bạn và nhấn "Compare & pull request". Điền thông tin:

  • Title: Mô tả ngắn gọn thay đổi (vd: feat: thêm gói libreoffice vào overlay)
  • Description: Mô tả chi tiết lý do, cách hoạt động, ảnh hưởng
  • Liên kết issue nếu có (vd: Closes #123)
💡 Mẹo: Tạo Draft PR nếu bạn muốn nhận phản hồi sớm trước khi hoàn thiện.

Quy tắc code

  • Ngôn ngữ: Nix, Bash, Go (tuỳ module)
  • Format: Dùng nix fmt cho Nix files
  • Không hardcode: Sử dụng NixOS module options thay vì hardcode giá trị
  • Bảo mật: Không commit mật khẩu, API key, hoặc thông tin nhạy cảm

Kiểm tra trước khi gửi PR

# Kiểm tra flake nix flake check # Build thử một gói nix build .#package-name # Kiểm tra cú pháp Nix nix eval .#checks.x86_64-linux # Chạy kiểm tra (nếu có) nix build .#checks.x86_64-linux.test

Sau khi PR được merge

Xin chúc mừng! Bạn đã trở thành người đóng góp cho BamOS 🎉. Hãy:

  • Cập nhật fork của bạn: git pull upstream main
  • Xoá nhánh cũ: git branch -d feat/ten-tinh-nang
  • Xoá remote branch: git push origin --delete feat/ten-tinh-nang
  • Thêm tên bạn vào CONTRIBUTORS.md (nếu muốn)
