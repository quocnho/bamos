🔑 Cách reset mật khẩu trên BamOS

Nếu bạn quên mật khẩu tài khoản người dùng trên BamOS, đừng lo lắng. Bạn có thể dễ dàng đặt lại mật khẩu bằng cách boot từ USB cài đặt và truy cập vào ổ cứng.

⚠️ Lưu ý: Các bước này yêu cầu quyền truy cập vật lý vào máy tính. Bạn không thể reset mật khẩu từ xa nếu không có quyền root.

Những gì bạn cần

  • Một USB boot chứa ISO BamOS (hoặc bất kỳ bản phân phối Linux nào)
  • Quyền truy cập vật lý vào máy tính
  • Kiến thức cơ bản về dòng lệnh

Các bước thực hiện

Bước 1: Boot từ USB cài đặt

Cắm USB boot BamOS vào máy, khởi động và chọn boot từ USB trong BIOS/UEFI (thường nhấn F12, F2, Del hoặc Esc tuỳ theo hãng máy).

Chọn tùy chọn "Try BamOS" hoặc "Live Environment" để vào môi trường sống (Live), không cài đặt.

Bước 2: Mở terminal

Sau khi boot vào môi trường Live, mở terminal (thường là Ctrl + Alt + T).

Bước 3: Tìm phân vùng hệ thống

Xác định ổ cứng chứa BamOS đã cài:

# Liệt kê các ổ đĩa lsblk # Kết quả sẽ hiển thị: # NAME MAJ:MIN RM SIZE RO TYPE MOUNTPOINTS # sda 8:0 0 238.5G 0 disk # ├─sda1 8:1 0 512M 0 part /boot/efi # ├─sda2 8:2 0 4G 0 part [SWAP] # └─sda3 8:3 0 234G 0 part /var

Trên hệ thống OSTree (BamOS sử dụng OSTree), phân vùng gốc thường được mount tại /var. Bạn cần mount phân vùng chính (thường là /dev/sda3 hoặc tương tự).

Bước 4: Mount hệ thống

# Mount phân vùng gốc sudo mount /dev/sdX3 /mnt # Mount phân vùng EFI (nếu có) sudo mount /dev/sdX1 /mnt/boot/efi # Mount các pseudo-filesystem cần thiết for dir in dev proc sys; do sudo mount --rbind /$dir /mnt/$dir done

Thay /dev/sdX3 bằng phân vùng chính của bạn (xem từ lsblk).

Bước 5: Chroot vào hệ thống

# Chroot vào hệ thống sudo chroot /mnt /bin/bash # Sau khi chroot thành công, bạn sẽ thấy shell thay đổi

Bước 6: Reset mật khẩu

Sau khi đã chroot thành công, sử dụng lệnh passwd để đặt lại mật khẩu cho tài khoản của bạn:

# Reset mật khẩu cho người dùng hiện tại passwd <tên-người-dùng> # Ví dụ passwd bamos # Nhập mật khẩu mới hai lần khi được yêu cầu
💡 Mẹo: Nếu bạn không nhớ tên người dùng, gõ cat /etc/passwd để xem danh sách tài khoản.

Bước 7: Thoát và khởi động lại

# Thoát chroot exit # Umount tất cả sudo umount -R /mnt # Khởi động lại máy và boot từ ổ cứng sudo reboot

Phương pháp thay thế: Sử dụng mật khẩu tạm thời

Nếu bạn vẫn còn quyền truy cập sudo (chưa quên mật khẩu của người dùng khác), bạn có thể đặt lại mật khẩu trực tiếp:

# Dùng sudo để đổi mật khẩu cho người dùng khác sudo passwd <tên-người-dùng>

Trường hợp đặc biệt: Tài khoản bị khoá

Nếu tài khoản bị khoá, bạn cần mở khoá trước khi đặt lại mật khẩu:

# Mở khoá tài khoản sudo passwd -u <tên-người-dùng> # Hoặc dùng usermod sudo usermod -U <tên-người-dùng>

Lưu ý quan trọng

  • Mật khẩu có phân biệt chữ hoa chữ thường. Hãy chắc chắn rằng Caps Lock đã tắt.
  • Mật khẩu nên có độ dài tối thiểu 8 ký tự, bao gồm chữ hoa, chữ thường, số và ký tự đặc biệt.
  • Sau khi reset, hãy đăng nhập ngay để kiểm tra.
  • Nếu bạn sử dụng mã hoá ổ đĩa (LUKS), bạn cần mở khoá ổ đĩa trước khi mount: sudo cryptsetup luksOpen /dev/sdX3 bamos-root
