🛠 Tự tạo ISO BamOS tùy chỉnh

BamOS là một bản phân phối mã nguồn mở dựa trên Nix, vì vậy bạn hoàn toàn có thể tự build ISO của riêng mình với những tuỳ chỉnh theo ý muốn. Hướng dẫn này dành cho những người dùng muốn tạo phiên bản BamOS riêng.

⚠️ Yêu cầu: Bạn cần có kiến thức cơ bản về Git, GitHub, Nix và Nix flakes. Một máy tính Linux với ít nhất 8 GB RAM và 40 GB dung lượng trống để build.

1. Fork repository BamOS trên GitHub

Đầu tiên, truy cập github.com/quocnho/bamos và fork repository về tài khoản GitHub của bạn.

# Clone repository đã fork về máy git clone https://github.com/<tên-bạn>/bamos.git cd bamos

2. Cấu trúc repository

Repository BamOS sử dụng cấu trúc Nix flake tiêu chuẩn. Các tệp và thư mục quan trọng bạn cần biết:

Tệp / Thư mục Mô tả
flake.nix Định nghĩa các gói, module, ISO build
iso/ Cấu hình build ISO cho từng edition
modules/ NixOS module tùy chỉnh của BamOS
pkgs/ Các gói phần mềm tùy chỉnh
overlays/ Overlay cho nixpkgs
profiles/ Cấu hình profile (Desktop, Gaming, Server)

3. Tuỳ chỉnh flake.nix

Mở tệp flake.nix và sửa đổi các thông số bạn muốn:

{ description = "BamOS — Custom Build"; inputs = { nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable"; bazzite.url = "github:ublue-os/bazzite"; }; outputs = { self, nixpkgs, ... } @ inputs: { # Thêm gói mới của bạn packages.x86_64-linux.my-app = nixpkgs.legacyPackages.x86_64-linux.callPackage ./pkgs/my-app {}; # Custom ISO iso = import ./iso/custom.nix { inherit inputs; }; }; }

Thêm gói mặc định vào ISO

Để thêm một gói vào ISO tùy chỉnh, sửa tệp cấu hình ISO trong thư mục iso/:

{ config, pkgs, ... }: { # Thêm gói vào system packages environment.systemPackages = with pkgs; [ vim git htop btop lazygit # gói bạn muốn thêm ]; # Kích hoạt service services.openssh.enable = true; }

4. Build ISO

Sau khi đã tuỳ chỉnh xong, bạn có thể build ISO với lệnh:

# Build ISO cho edition Desktop nix build .#iso-desktop # Build ISO cho edition Gaming nix build .#iso-gaming # Build ISO cho edition Server nix build .#iso-server # Build ISO cho edition Minimal nix build .#iso-minimal

Sau khi build hoàn tất, tệp ISO sẽ nằm trong thư mục result/.

⏱ Lưu ý: Lần build đầu tiên có thể mất rất lâu (30–60 phút) vì Nix phải tải và build tất cả dependencies. Các lần sau sẽ nhanh hơn nhờ cache.

5. Ghi ISO vào USB

Sử dụng một trong các công cụ sau để ghi ISO vào USB:

# Dùng dd (Linux) sudo dd if=result/bamos-*.iso of=/dev/sdX bs=4M status=progress # Dùng rufus (Windows) # Dùng balenaEtcher (macOS / Windows / Linux) # Dùng Fedora Media Writer sudo mediawriter --write result/bamos-*.iso

6. Đẩy lên GitHub và chia sẻ

Sau khi build thành công, bạn có thể commit và push lên fork của mình, hoặc tạo Pull Request đóng góp lại cho BamOS:

git add . git commit -m "feat: thêm gói my-app vào ISO" git push origin main

Xử lý lỗi thường gặp

Lỗi Giải pháp
out of memory Tăng swap hoặc chạy trên máy có RAM ≥ 8 GB
hash mismatch Cập nhật hash với nix flake update
build failed Kiểm tra lỗi ở log, thường do thiếu dependency
📖 Tham khảo thêm: Đọc hướng dẫn đóng góp nếu bạn muốn gửi thay đổi về repository chính thức.
